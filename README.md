# SceneForge v1 - 基础剧本编辑器

## 版本概述

SceneForge 的第一个版本，实现了剧本写作工具的核心骨架：项目结构搭建、暗色主题样式系统、本地存储模块，以及支持标题与正文编辑的基础编辑器。

## 功能清单

- **剧本标题编辑** — 顶部输入框，支持最多120字符的剧本标题
- **正文内容编辑** — 等宽字体文本区域，支持自由编写剧本内容
- **自动保存** — 输入停止1.5秒后自动将内容保存到 localStorage
- **保存状态指示** — 底部状态栏实时显示保存进度（保存中 / 已保存）
- **字符计数** — 底部实时统计正文字符数
- **数据持久化** — 页面刷新或关闭后重新打开，自动恢复上次编辑内容
- **页面关闭前保存** — 监听 beforeunload 事件，确保不丢失数据

## 技术栈

- 纯 HTML/CSS/JS，无需构建工具，直接用浏览器打开
- localStorage 实现客户端持久化
- IIFE 模块模式组织代码
- CSS 自定义属性实现暗色主题

## 文件结构

```
v1-basic-editor/
├── index.html          # 主页面
├── css/
│   └── style.css       # 样式表（暗色主题）
├── js/
│   ├── storage.js      # 存储模块（localStorage 封装）
│   └── app.js          # 主应用模块（编辑器逻辑）
└── README.md
```

## 使用方式

1. 直接用浏览器打开 `index.html`
2. 在顶部输入剧本标题
3. 在正文编辑区编写剧本内容
4. 所有修改会自动保存，刷新页面不丢失

# SceneForge v2 - 角色管理模块

## 版本概述

在 v1 基础编辑器之上，新增**角色管理**功能模块。通过右侧功能面板，用户可以创建、编辑、删除剧本角色，每个角色支持名称、类型（主角/反派/配角/次要角色）和文字描述。页面布局从单栏改为左右分栏结构，为后续场景、时间线等模块预留标签入口。

## 新增功能

- **角色CRUD** — 添加、编辑、删除角色
- **角色属性** — 名称（必填）、角色类型（下拉选择）、描述文字
- **角色列表** — 卡片式展示所有已创建角色，显示名称、类型标签、描述
- **侧边栏面板** — 右侧功能面板，使用标签切换不同功能模块
- **标签式导航** — 角色/场景/时间线/导出四个标签（后续版本逐步启用）
- **数据持久化** — 角色数据保存到 localStorage，刷新不丢失

## 相较于 v1 的变更

| 文件  | 变更类型 | 说明  |
| --- | --- | --- |
| `index.html` | 修改  | 重构为左右分栏布局，增加侧面板和角色UI |
| `css/style.css` | 扩展  | 新增面板、标签、表单、角色卡片、按钮等样式 |
| `js/storage.js` | 扩展  | 新增 CHARACTERS 存储键 |
| `js/characters.js` | 新增  | 角色管理模块（CRUD + 渲染） |
| `js/app.js` | 修改  | 增加Characters模块初始化和标签切换逻辑 |

## 技术要点

- 角色数据使用 JSON 数组序列化存储
- 角色ID使用时间戳+随机字符串生成，保证唯一性
- 表单内联在面板中，通过显示/隐藏切换
- 事件委托处理角色列表中的编辑/删除按钮点击

## 使用方式

1. 浏览器打开 `index.html`
2. 左侧编辑区正常编写剧本
3. 右侧面板「角色」标签下，点击「+ 添加角色」
4. 填写角色名称、选择类型、输入描述后保存
5. 角色卡片出现在列表中，支持编辑和删除

# SceneForge v3 - 场景分解模块

## 版本概述

在 v2 基础上新增**场景分解**功能模块。用户可以将剧本拆分为多个独立场景，每个场景包含标题、地点、时间、概要描述和出场角色。场景与角色模块联动——创建场景时可直接勾选已创建的角色作为出场人物。场景面板与角色面板通过标签切换。

## 新增功能

- **场景CRUD** — 添加、编辑、删除场景
- **场景属性** — 标题（必填）、地点、时间（日/夜/下午等）、概要描述
- **场景-角色关联** — 通过复选框选择已创建的角色作为出场角色
- **场景列表** — 按序展示所有场景，显示序号、标题、元信息（地点/时间/角色）
- **场景面板** — 右侧面板新增「场景」标签页
- **标签联动** — 角色和场景标签均可自由切换使用

## 相较于 v2 的变更

| 文件  | 变更类型 | 说明  |
| --- | --- | --- |
| `index.html` | 修改  | 新增场景面板UI，启用场景标签，引入scenes.js |
| `css/style.css` | 扩展  | 新增场景卡片、场景表单、复选框组、序号样式 |
| `js/storage.js` | 扩展  | 新增 SCENES 存储键 |
| `js/scenes.js` | 新增  | 场景管理模块（CRUD + 角色关联） |
| `js/characters.js` | 无变更 | 保持不变 |
| `js/app.js` | 修改  | 新增 Scenes 模块初始化 |

## 技术要点

- 场景按 `order` 字段排序展示
- 场景与角色通过 `characterIds` 数组关联
- 创建/编辑场景时动态读取 Characters.getAll() 渲染复选框
- 场景卡片使用绝对定位的编辑/删除按钮

## 使用方式

1. 先在「角色」面板创建剧本角色
2. 切换到「场景」面板，点击「+ 添加场景」
3. 填写场景标题、地点、时间、概要
4. 在「出场角色」区域勾选该场景中出场的角色
5. 保存后场景卡片出现在列表中

# SceneForge v4 - 时间线视图

## 版本概述

在 v3 基础上新增**故事时间线**可视化模块。用户可以在时间线面板中看到所有场景按幕结构排列的垂直时间线，支持场景上移/下移调整顺序、将场景分配至不同幕（Act I/II/III），并提供场景统计概览。时间线视图以连接线+节点方式展示故事结构，不同幕使用不同颜色区分。

## 新增功能

- **垂直时间线** — 以连接线+节点方式展示所有场景的排列顺序
- **幕结构管理** — 场景可分配到第一幕/第二幕/第三幕，时间线按幕分组显示
- **场景排序** — 上移/下移按钮调整场景在时间线中的顺序
- **幕内移动** — 通过下拉选择器将场景在幕之间移动
- **场景统计** — 顶部统计栏显示总场景数和各幕场景数量
- **颜色编码** — 第一幕蓝色、第二幕橙色、第三幕红色，一目了然
- **数据联动** — 排序结果同步回场景面板的列表

## 相较于 v3 的变更

| 文件  | 变更类型 | 说明  |
| --- | --- | --- |
| `index.html` | 修改  | 新增时间线面板，启用时间线和导出标签，引入timeline.js |
| `css/style.css` | 扩展  | 新增时间线风格样式：节点、连接线、幕标题、统计栏、颜色编码 |
| `js/timeline.js` | 新增  | 时间线模块：渲染、排序、幕分配 |
| `js/scenes.js` | 修改  | 场景数据新增 `act` 字段支持幕分配 |
| `js/app.js` | 修改  | 新增 Timeline 初始化，切换时间线时刷新，刷新按钮绑定 |

## 技术要点

- 场景数据新增 `act` 字段（'1'/'2'/'3'），默认 '1'
- 时间线通过读取 Scenes.getAll() 并按 act 分组渲染
- 场景排序通过交换 order 字段实现
- 幕切换通过下拉选择器直接修改 scene.act 并重新渲染
- 模块间解耦：Timeline 通过公开 API（Scenes.getAll, Characters.getById）读取数据

## 使用方式

1. 先在「角色」面板创建角色，在「场景」面板创建场景
2. 切换到「时间线」面板，查看自动生成的时间线
3. 点击场景旁的 ▲/▼ 按钮调整场景顺序
4. 使用下拉选择器将场景切换到不同幕
5. 点击「刷新」按钮手动刷新时间线视图
